mkdir proj

cd src

cd ..

Webpack会帮助我们生成dist目录。

初始化工程

npm init

安装依赖

npm install -g webpack
npm install –save react react-dom @types/react @types/react-dom

  • 使用@types/前缀表示我们额外要获取React和React-DOM的声明文件。 通常当你导入像 “react”这样的路径，它会查看react包； 然而，并不是所有的包都包含了声明文件，所以TypeScript还会查看 @types/react包。 你会发现我们以后将不必在意这些。

npm install –save-dev typescript awesome-typescript-loader source-map-loader

  • awesome-typescript-loader可以让Webpack使用TypeScript的标准配置文件 tsconfig.json编译TypeScript代码。
    source-map-loader使用TypeScript输出的sourcemap文件来告诉webpack何时生成自己的sourcemaps。 这就允许你在调试最终生成的文件时就好像在调试TypeScript源码一样。

Git提交规范

目前，我们使用了commitizen工具集来规范commit提交，遵循Angular团队的规范，并进行了定制化配置。

格式规范： type(scope): subject

  1. 我们可以使用git commit命令，不过提交描述要符合上述书写规范，否则无法通过校验。

    type和subject是必需，scope是可选，例如：
    git commit -m ‘feat(containers): 提交新功能’
    git commit -m ‘feat: 提交新功能’

  2. 我们可以通过npx git cz或npm run commit命令进行选择方式提交。

  3. 我们还可以全局安装commitizen，使用git cz命令进行选择方式提交。

注意：
type必须是范围内的一个，
scope是我们修改的模块或者说影响的范围，建议填写模块名。冒号后要有一个空格。
subject是短说明。
